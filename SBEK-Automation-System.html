01 System Overview

SBEK Automation is an end-to-end operations and marketing automation platform built for SBEK Jewelry. It automates everything from the moment a customer places an order to social media posting of product creatives.

Services
5
App, Dashboard, Crawler, PostgreSQL, Redis
Job Queues
7
Order sync, notifications, content, creatives, social, crawl, reviews
Cron Jobs
4
Daily sync, daily reviews, weekly content, weekly crawl
External APIs
10+
WooCommerce, Sheets, Drive, OpenAI, Gemini, WhatsApp, Email, Postiz

What It Automates

AreaWhat Happens
Order ProcessingWooCommerce order → synced to Google Sheets (Orders, Production, QC tabs) → owner notified via WhatsApp + Email
Customer CommsAuto-send order confirmations, shipping updates, and review requests via WhatsApp (Meta API) + Email (SMTP)
AI ContentNew product published → AI generates SEO meta, FAQ, comparison article, schema markup, internal links (OpenAI GPT-4o)
AI CreativesProduct images auto-generated in 5 variants: white bg, lifestyle, festive, minimal text, story format (Google Gemini)
Social MediaGenerated content + creatives auto-posted to Instagram, Facebook, LinkedIn, Twitter, Pinterest (Postiz)
Competitor IntelWeekly headless browser crawl of competitor sites → tracks product/price changes, blog activity, SEO health
Review Collection5 days after delivery → customer gets WhatsApp + Email asking for a review

Internet (WooCommerce, Users) │ ▼ ┌──────────────────────────────────────────────────────┐ │ APP (Express API — port 3000) │ │ │ │ Routes: │ │ /webhooks/woocommerce/* ← receives orders │ │ /dashboard/* ← serves dashboard API │ │ /admin/settings/* ← admin config │ │ /health ← health checks │ │ │ │ Workers (BullMQ): │ │ order-sync · notification · review-request │ │ content-generation · creative-generation │ │ social-posting · competitor-crawl │ │ │ │ Cron (node-cron): │ │ 2AM daily sync · 6AM reviews · Mon content · Sun crawl│ └───────┬──────────┬───────────┬───────────────────────┘ │ │ │ ▼ ▼ ▼ PostgreSQL Redis Crawler 5 tables 7 queues Playwright port 5432 port 6379 port 3001 ▲ │ ┌───────┴──────────────────────────────────────────────┐ │ DASHBOARD (Next.js — port 3002) │ │ │ │ Proxies /api/* → App │ │ Pages: Home · Queues · Activity · System · Settings │ └──────────────────────────────────────────────────────┘
Key principle: Only the App talks to PostgreSQL, Redis, and external APIs. The Dashboard only talks to the App. The Crawler only runs when the App's competitor-crawl worker calls it. PostgreSQL and Redis are never exposed to the internet.

07 Cron Jobs (Scheduled Tasks)

All cron jobs run via node-cron in the App service. Timezone: Asia/Kolkata (IST). Each execution is logged in the cron_runs table.

1
Daily Sheets Sync — 2:00 AM IST
Pulls last 3 days of orders from WooCommerce API (paginated, 50/page). Enqueues an order-sync job for each order. Purpose: catches any missed webhooks for reconciliation.
WooCommerce API → order-sync queue → Google Sheets
2
Daily Review Requests — 6:00 AM IST
Reads Google Sheets for orders with status "Delivered". Filters for orders delivered 5+ days ago where Notes column doesn't contain "Review Sent". Enqueues review-request job per order. Updates Notes to "Review Sent [date]" to prevent duplicates.
Google Sheets → review-request queue → WhatsApp + Email
3
Weekly Content Generation — Monday 9:00 AM IST
Fetches all published products from WooCommerce. For each product, enqueues 6 content-generation jobs + 1 creative-generation job = 7 jobs per product. If you have 27 products, that's 189 jobs.
WooCommerce API → content-generation queue + creative-generation queue → OpenAI + Gemini
4
Weekly Competitor Crawl — Sunday 10:00 PM IST
Reads competitor list from Google Sheets "Competitors" tab. Enqueues one competitor-crawl job per competitor. Each job calls the Crawler microservice which launches headless Chromium.
Google Sheets → competitor-crawl queue → Crawler (Playwright) → PostgreSQL

09 Crawler Deep Dive

A separate microservice that runs headless Chromium to scrape competitor jewelry websites. Built with Playwright 1.49.0 + Cheerio 1.0.0.

Browser Configuration

SettingValue
BrowserChromium (headless, no GUI)
User AgentChrome 120 on Windows 10 (realistic, avoids bot detection)
Viewport1920 × 1080
Navigation timeout30 seconds
Post-load wait2 seconds (for JavaScript rendering)
Polite delay2-3 seconds random between page navigations

Crawl Process (Step by Step)

1
Check robots.txt
Fetches /robots.txt, parses Disallow rules. If crawling is blocked → returns error, doesn't proceed.
2
Launch Chromium & Navigate
Opens headless browser → navigates to homepage → waits 2s for JavaScript to fully render.
3
Extract Homepage Metadata
Title, meta description, all H1 tags, meta keywords, og:image, navigation links from <nav>/<header> elements.
4
Calculate SEO Score
4 boolean checks: has meta description? has H1? has Open Graph tags? has JSON-LD structured data?
5
Extract Products (3 strategies)
Strategy 1: CSS selectors for product cards (23 name selectors, 15 price selectors). Strategy 2: Direct product links (/products/, /shop/). Strategy 3: JSON-LD schema.org Product data. If no products on homepage → follows links to /collections or /shop page.
6
Extract Blog Posts (2 strategies)
Strategy 1: Blog entry container selectors (11 selectors). Strategy 2: URL pattern matching (/blog/, /journal/, /news/). If no posts on homepage → follows link to blog page.
7
Compare with Previous Crawl
Detects new/removed products, price changes >10%, new blog posts. Calculates content freshness score (0-100).
8
Generate HTML Report
Creates a styled HTML report with summary cards, SEO health grid, product/price change tables, freshness bar. Saved to /app/reports/.

Docker Image

Base: mcr.microsoft.com/playwright:v1.49.0-noble (Microsoft's official Playwright image with Chromium pre-installed). This is a ~1.3GB image because it includes a full browser engine.
