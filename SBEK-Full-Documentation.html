05 Crawler Microservice — Complete Internals

Location: crawler/ · Tech: Express + Playwright + Cheerio · Docker: mcr.microsoft.com/playwright:v1.49.0-noble (~1.3 GB) · Port: 3001

Browser Configuration

SettingValueWhy

Crawl Process (8 Steps)

1
robots.txt check
Fetches {site}/robots.txt (5s timeout). Parses User-agent: * sections. If path is in Disallow rules → returns error, doesn't crawl.
2
Launch Chromium + navigate
Opens headless browser → navigates to URL (30s timeout) → waits 2s for JS rendering
3
Extract homepage metadata
title, meta description, all H1 tags, meta keywords, og:image, navigation links from <nav>/<header>/.main-nav
4
Calculate SEO score
4 booleans: hasMetaDescription, hasH1, hasOgTags (og:title or og:image), hasStructuredData (JSON-LD scripts)
5
Extract products (3 strategies)
S1: 23 CSS selectors for product names + 15 for prices + 8 for links. Finds parent card container, extracts name/price/URL.
S2: Direct product links matching /products/, /product/, /shop/ patterns.
S3: JSON-LD schema.org Product/@type structures.
If no products found on homepage → follows link to /collections or /shop page (with polite delay).
6
Extract blog posts (2 strategies)
S1: 11 CSS selectors for blog containers (.blog-post, .article-card, etc.).
S2: URL patterns matching /blog/, /journal/, /news/, /articles/, /stories/.
If none on homepage → follows link to blog page.
7
Compare with previous crawl
New/removed products (case-insensitive name match). Price changes >10% threshold. New blog posts (URL-based match). Content freshness score 0-100.
8
Generate HTML report
Styled report with: summary cards, SEO health grid (4 checkmarks), freshness bar (color-coded), product/price change tables, full product list, blog posts, navigation. Saved to /app/reports/{date}-{domain}.html

Customer buys on sbek.in │ ▼ WooCommerce fires POST /webhooks/woocommerce/order │ (HMAC-SHA256 verified) ▼ App logs to webhook_events table │ ▼ Enqueues to order-sync queue (Redis) │ ▼ Worker calls WooCommerce API for full order │ ├─→ Google Sheets: append Orders, Production, QC tabs ├─→ Google Sheets: upsert Customers tab └─→ Enqueue notification job │ ├─→ WhatsApp to owner (Meta → Wati → Interakt) └─→ Email to customer (SMTP + Handlebars) │ ▼ job_logs table updated: status=completed

Flow B: Product Published

Product published on WooCommerce │ ▼ POST /webhooks/woocommerce/product │ ▼ App auto-creates 7 jobs: │ ├─→ content-generation ×6: │ SEO meta → OpenAI → WooCommerce meta │ FAQ → OpenAI → WooCommerce meta (JSON-LD) │ AEO KB → OpenAI → WordPress page │ Comparison→ OpenAI → WordPress blog post │ Schema → OpenAI → WooCommerce meta │ Links → WooCommerce → product description │ └─→ creative-generation ×1: 5 variants → Gemini/DALL-E → Sharp resize │ ▼ Google Drive upload → Google Sheets log │ ▼ social-posting → Postiz → 5 platforms

Flow C: Competitor Monitoring

Sunday 10 PM IST → cron fires │ ▼ Reads Google Sheets Competitors tab │ ▼ Enqueues competitor-crawl per competitor │ ▼ Worker → POST crawler:3001/crawl │ ▼ Playwright launches Chromium ├─→ robots.txt check ├─→ Navigate homepage (30s timeout) ├─→ Wait 2s for JS ├─→ Extract: products, prices, blog, SEO ├─→ Follow links if needed (polite 2-3s delay) └─→ Compare with previous crawl │ ▼ Save to competitor_snapshots table Generate HTML report OpenAI enhanced analysis Alert via WhatsApp if significant

Graceful Shutdown (src/index.ts)

On SIGTERM or SIGINT: close HTTP server → close all BullMQ queues → quit Redis → end PostgreSQL pool → process.exit(0). Unhandled rejections logged as error. Uncaught exceptions logged as fatal + exit(1).
